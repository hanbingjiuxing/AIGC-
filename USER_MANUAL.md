# AIGC社信息系统 - 操作手册 (User Manual)

欢迎使用 AIGC社信息系统！本手册将引导您了解如何部署环境、启动、使用及维护该系统。

---

## 🛠️ 1. 环境部署 (First Time Setup)

如果您是第一次在当前电脑上运行该系统，请先执行环境部署脚本：

1.  **一键部署环境**：双击项目根目录下的 `setup_env.bat`。
    *   该脚本会自动：
        *   检查 Python 和 Node.js 环境。
        *   创建 Python 虚拟环境并安装后端依赖库。
        *   通过 npm 安装前端所有必要的插件。
    *   **注意**：运行过程中请保持网络畅通。

---

## 🚀 2. 系统启动指南

系统已配置一键启动脚本，无需手动输入复杂指令。

1.  **一键启动**：双击项目根目录下的 `run_system.bat`。
    *   该脚本会自动启动后端服务（Flask）和前端服务（Vite）。
    *   启动完成后，系统会自动在您的默认浏览器中打开 `http://localhost:5173`。
2.  **局域网访问**：若需在其他设备访问，请查看 `DEPLOY_LAN.md` 获取 IP 配置信息。

---

## 👤 3. 学生用户手册

作为社员，您可以进行日常打卡、作品提交和查看公告。

### 2.1 仪表盘 (Dashboard)
*   **公告栏**：登录后首页即可看到社团最新动态。
*   **入社天数/统计**：右侧展示您的个人入社时光及各项数据统计。

### 2.2 作品管理 (Works)
*   **提交作品**：点击“上传作品”按钮，填写标题、深度解析并选择文件即可发布。
*   **查看详情**：点击列表中的作品项，可弹出“毛玻璃”详情浮窗，查看自己填写的描述并支持下载源码。
*   **附件下载**：点击列表右侧的下载图标，文件将以您上传时的**原始文件名**保存。

### 2.3 签到打卡 (Attendance)
*   点击侧边栏“签到打卡”，点击“立即签到”即可完成今日考勤。系统会自动统计您的累计打卡次数。

---

## 👨‍🏫 4. 教师/社长管理手册

管理人员额外拥有成员管理和公告发布的权限。

### 3.1 成员管理 (Member Management)
*   **查看列表**：侧边栏进入“成员管理”，可实时查看所有社员名单及其入社时间。
*   **查看作品**：点击社员右侧的“查看作品”，可查看该社员提交的所有历史记录，并支持**一键下载**其作品。
*   **信息修改**：点击“编辑”，可修改社员的姓名、密码或调整其职位。

### 3.2 公告发布 (Announcements)
*   在个人资料 (Profile) 页面，管理人员拥有“管理公告”的权限。
*   支持发布新公告、修改现有公告或删除失效公告。

---

## 🛠️ 5. 管理员后台工具 (GUI Tool)

系统提供了一个独立的图形化后台工具，用于处理紧急情况。

*   **启动方式**：在 `backend` 目录下运行 `python admin_gui.py`。
*   **核心功能**：
    *   **用户管理**：快速添加新用户、重置任意用户密码、批量删除用户。
    *   **公告维护**：直接操作数据库层面的公告记录。
    *   **清理统计**：一键清空考勤记录或删除已选中的作品记录。

---

## 📂 6. 存储与数据

*   **用户数据 (需保留)**：
    *   `backend/instance/aigc_society.db`：数据库文件，存储所有账号密码、签到记录、作品描述等。
    *   `backend/uploads/`：文件夹，存储所有学生提交的作品附件。
*   **程序数据 (可覆盖)**：
    *   `frontend/` 文件夹：所有前端界面代码。
    *   `backend/` 中的 `.py` 文件及子文件夹：所有后端业务逻辑。
    *   根目录 `.bat` 脚本：启动和部署脚本。

---

## 🔄 7. 系统更新指南 (覆盖升级)

当您需要使用新版本的代码覆盖旧版本时，请务必遵循以下步骤以防止数据丢失：

1.  **备份**：将整个项目文件夹完整复制一份作为备份。
2.  **提取核心数据**：从小旧项目文件夹中复制 `backend/instance/aigc_society.db` 和 `backend/uploads/` 文件夹到桌面。
3.  **覆盖安装**：将新版本的代码直接解压并覆盖到您的项目目录中。
4.  **数据还原**：将刚才复制到桌面的 `.db` 文件和 `uploads/` 文件夹放回新版本的 `backend/instance`（数据库）和 `backend`（各文件夹）相应目录下。
5.  **启动**：双击 `run_system.bat` 运行。

---
